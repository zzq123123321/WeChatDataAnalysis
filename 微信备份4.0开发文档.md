# WeChatDataAnalysis 开发文档

## 项目概述

微信 4.x 数据库解密分析工具。后端 Python/FastAPI 提供 API，前端 Vue/Nuxt 渲染，Electron 打包桌面应用。

## 构建安装包

```powershell
cd D:\1\desktop
npm run dist
```

分步执行：
1. `build:ui` — 生成前端静态文件 → `desktop/resources/ui/`
2. `build:backend` — PyInstaller 打包 → `desktop/resources/backend/wechat-backend.exe`
3. `build:icon` — 生成图标
4. `electron-builder --win --x64` — 打包 NSIS 安装包 → `desktop/dist/*-Setup.exe`

安装包输出：`D:\1\desktop\dist\WeChatDataAnalysis-{version}-Setup.exe`

> **注意**：构建受 GitHub 网络影响。electron-builder 需要下载 `winCodeSign`（用于 rcedit 设置 exe 版本/图标）。已配置镜像：
> ```powershell
> $env:ELECTRON_BUILDER_BINARIES_MIRROR = "https://npmmirror.com/mirrors/electron-builder-binaries/"
> ```

## 后端 API 架构

FastAPI 应用，路由统一前缀 `/api`。

### 核心 API 端点

| 端点 | 方法 | 说明 |
|---|---|---|
| `/api/decrypt/start` | POST | 开始解密 |
| `/api/decrypt/status` | GET | 解密状态 |
| `/api/keys/list` | GET | 密钥列表 |
| `/api/chat/conversations` | GET | 会话列表 |
| `/api/chat/messages` | GET | 聊天消息 |
| `/api/nas/sync` | POST | 触发 NAS 同步 |
| `/api/nas/sync/status` | GET | 同步进度状态 |
| `/api/nas/sync/cancel` | POST | 取消同步 |
| `/api/nas/auto-sync` | GET/POST | 自动同步配置 |
| `/api/system/settings` | GET/POST | 系统设置 |

### NAS 同步模块 (`nas_storage.py`)

使用 **paramiko** (SSH/SFTP) 替代旧的 sshpass/ssh/scp 命令行。

核心函数：
- `NASStorage.connect()` — SSH + SFTP 连接
- `NASStorage.scan_remote_files()` — 递归遍历远程目录
- `NASStorage.sync_databases_to_nas(account="")` — 增量同步
- `NASStorage.cancel_sync()` — 取消同步

同步流程：
1. 扫描本地 `output/` 目录获取所有 `.db` 文件
2. 按 `account/` 前缀过滤（如果指定了账号）
3. 扫描远程目录获取已有文件信息
4. 对比 mtime + 文件大小，找出需要上传的文件
5. 逐个上传，更新进度

进度跟踪使用全局字典 `_sync_progress`，包含：
```python
{
    "running": bool,
    "current_file": str,
    "current_index": int,
    "total_files": int,
    "percentage": int,
    "message": str
}
```

取消机制：设 `running=False`，线程在下次上传前检查退出。

### 数据目录

- 输出目录：`%APPDATA%/wechat-data-analysis-desktop/output/`
- 运行时设置：同目录下 `runtime_settings.json`
- NAS 密码：同目录下 `nas_password.dat`（加密存储）

## 常见任务

### 添加新 API 端点

1. 在 `src/wechat_decrypt_tool/routers/` 下创建或修改路由文件
2. 使用 `@router.get/post` 装饰器
3. 在 `api.py` 中 `app.include_router(router)`
4. 前端调用：`axios.get('/api/...')`

### 修改前端状态

使用 Pinia store：
```javascript
// 在组件中
import { useChatAccountsStore } from '~/stores/chatAccounts'
const store = useChatAccountsStore()
store.selectedAccount = '...'
```

### 设置环境变量

```powershell
# 后端端口
$env:WECHAT_TOOL_PORT = "10392"

# 前端 API 地址
$env:NUXT_PUBLIC_API_BASE = "http://127.0.0.1:10392/api"

# electron-builder 镜像（国内）
$env:ELECTRON_BUILDER_BINARIES_MIRROR = "https://npmmirror.com/mirrors/electron-builder-binaries/"

# 跳过代码签名
$env:CSC_IDENTITY_AUTO_DISCOVERY = "false"
$env:CSC_LINK = ""
```

### 调试 NAS 同步

```powershell
uv run uvicorn src.wechat_decrypt_tool.api:app --reload --port 10391 --log-level debug
# 访问 http://localhost:3000 → 设置 → NAS 同步
```

直接测试同步（不通过前端）：

```powershell
# 触发同步
curl -X POST http://127.0.0.1:10391/api/nas/sync

# 查看状态
curl http://127.0.0.1:10391/api/nas/sync/status

# 取消同步
curl -X POST http://127.0.0.1:10391/api/nas/sync/cancel
```

### PyInstaller 注意事项

- 打包配置在 `desktop/scripts/build-backend.cjs`
- 需要显式指定 `hiddenimports` 确保模块被打包（如 `paramiko`、`bCrypt` 等）
- 打包后 exe 路径：`desktop/resources/backend/wechat-backend.exe`
- 启动时自动解压到临时目录运行
